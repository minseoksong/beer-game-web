# 🍺 맥주 분배 게임 (Beer Distribution Game)

[English](./README.md) · **한국어**

MIT Sloan에서 1960년대 Jay Forrester가 시스템 다이내믹스와 공급사슬 교육용으로 개발한 **Beer Distribution Game**의 현대적 웹 구현체입니다. 이 저장소에는 두 가지 구현이 들어 있습니다 — 빠른 시연용 단일 파일 버전과, 수업용 풀스택 멀티팀 웹앱.

게임의 목적은 학생들이 공급사슬의 **채찍효과(Bullwhip Effect)** — 주문 신호의 변동성이 상류로 갈수록 증폭되어 시스템 전체를 비효율적으로 만드는 현상 — 을 직접 체험하면서 배우게 하는 것입니다.

---

## 📂 이 저장소 구성

### 1️⃣ 단일 HTML 파일 — [`beer_game.html`](./beer_game.html)

**파일 더블클릭**만으로 실행되는 의존성 0 버전. 한 사람이 4역할(소매상 / 도매상 / 유통업자 / 공장)을 모두 진행하거나, 일부 자리를 AI가 채우게 할 수 있습니다. 설치, 서버, 빌드 모두 불필요.

**적합한 용도:** 빠른 데모, 개인 학습, 1대1 시연.

### 2️⃣ 멀티팀 웹앱 — [`beer-game-web/`](./beer-game-web/)

여러 팀이 각자 다른 컴퓨터/스마트폰에서 동시에 진행하는 풀스택 웹앱. 강사가 6자리 세션 코드를 발급하면 학생들이 각자 접속해 자기 팀과 역할을 선택해서 실시간으로 함께 플레이합니다.

- **서버:** Node.js + Express + Socket.io + JSON / SQLite 저장소
- **클라이언트:** React + Vite + Tailwind + Chart.js
- **AI 정책 4종**: Base-Stock, Naive, Conservative, Reactive
- **수요 패턴 10종**: MIT 클래식 계단형부터 팬데믹 급증까지
- **시나리오 프리셋 5종**: Classic, Pandemic, Holiday, Stable, Chaos
- **한국어 / 영어** UI (브라우저 언어 자동 감지)
- **Docker 한 줄 배포** + 선택적 Caddy 자동 HTTPS

**적합한 용도:** 수업, 세미나, 다수 팀 동시 진행 워크숍.

자세한 사용법은 [`beer-game-web/README.md`](./beer-game-web/README.md) 참고.

## 🚀 빠른 시작

### 단일 HTML 게임 시도

```
beer_game.html 파일 더블클릭
```

이게 끝입니다. 브라우저에서 열리고 클라이언트 사이드에서만 동작.

### 멀티팀 웹앱 실행

Docker 사용 시:

```bash
cd beer-game-web
docker compose up -d
```

브라우저에서 `http://localhost:3001/` 접속.

Docker 없이 개발 모드로(코드 변경 시 핫 리로드):

```bash
# 터미널 1 — 서버
cd beer-game-web/server && npm install && npm run dev

# 터미널 2 — 클라이언트
cd beer-game-web/client && npm install && npm run dev
```

브라우저에서 `http://localhost:5173/` 접속.

## 🎯 게임 구조

```
고객 → [소매상] → [도매상] → [유통업자] → [공장]
        ↑          ↑           ↑           ↑
        주문이 상류로 흐름. 주문 지연 2주 + 배송 지연 2주.
```

각 단계는 자기 인접 단계만 볼 수 있고 매주 상류에 얼마나 주문할지 결정해야 합니다. 시스템에 지연이 있고 정보가 제한적이라, 작은 고객 수요 변동도 공장 단계에 가서는 거대한 변동으로 증폭됩니다. 학생들은 이를 직접 겪으면서 다음을 배웁니다:

- 단계 간 **정보 격리**의 비용
- **리드타임 지연**이 시스템 동학을 불안정하게 만드는 방식
- **일관된 주문 정책**(예: base-stock)의 가치
- 현실에서 Walmart-P&G의 VMI, Dell의 build-to-order, Zara의 Quick Response 같은 해법이 존재하는 이유

## 📈 결과 예시 (36주, 모두 AI base-stock 정책)

| 단계 | 주문 표준편차 | 채찍효과 증폭 | 누적 비용 |
|------|---------------|---------------|-----------|
| 고객 수요 | 1.26 | 1.00× (기준) | — |
| 소매상 | 10.37 | **8.25×** | $449 |
| 도매상 | 22.80 | **18.14×** | $2,269 |
| 유통업자 | 47.18 | **37.53×** | $5,362 |
| 공장 | 74.99 | **59.65×** | $5,770 |

원래 변동성 1.26이 공장에서 약 75로 증폭 — **60배 증폭**. 이게 시각화된 채찍효과입니다.

## 🛠 아키텍처

웹앱은 단일 Node.js 프로세스가 API와 빌드된 React 클라이언트를 동시에 서빙하고, Socket.io를 통한 WebSocket 실시간 동기화로 동작합니다. 기본 저장소는 외부 의존성 없는 JSON 파일이며, 큰 규모 배포를 위해 SQLite도 선택적으로 사용 가능합니다. 게임 상태는 모두 서버에서 관리되고 클라이언트는 렌더링만 담당해서 — 부정행위 방지와 재접속 시 일관성 보장.

깊이 있는 아키텍처 개요, 설계 근거, 6단계 개발 로드맵은 [`PLAN.md`](./PLAN.md) 참고.

## 🧪 테스트

```bash
cd beer-game-web/server
npm test
```

Node 내장 테스트 러너로 단위/회귀 테스트 32개 — 외부 테스트 의존성 없음. 포함 항목:

- 4가지 AI 정책 모두 동작 확인
- 10가지 수요 패턴 sanity 체크
- 단일 HTML 버전과의 회귀 테스트 (동일 출력)
- 5가지 시나리오 프리셋 끝까지 진행

## 🌍 배포

단일 호스트 수업용은 `docker compose up -d`로 충분합니다. 도메인이 있고 외부 노출이 필요한 경우:

```bash
DOMAIN=beer-game.example.com docker compose -f docker-compose.production.yml up -d
```

Caddy 리버스 프록시 뒤에 앱을 띄우고 Let's Encrypt SSL 인증서를 자동 발급/갱신합니다. 자세한 내용은 [`beer-game-web/README.md`](./beer-game-web/README.md) 참고.
